## 🎯 시스템 개요

실시간 진행률 추적 시스템은 계층적 평가 과정에서 사용자의 진행 상황을 실시간으로 모니터링하고 시각적으로 표시하는 시스템입니다.

### 핵심 기능
1. **실시간 진행률 모니터링**
2. **WebSocket 기반 양방향 통신**
3. **예상 완료 시간 계산**
4. **단계별 일관성 상태 추적**
5. **시각적 진행률 표시**

## 🏗️ 시스템 아키텍처

### 컴포넌트 구조
```
ProgressTracker
├── WebSocket 연결 관리
├── 진행률 메트릭 계산
├── 시각적 진행률 표시
└── 사용자 인터랙션 처리
```

### 데이터 플로우
```
사용자 평가 → WebSocket → ProgressTracker → UI 업데이트
     ↓
진행률 계산 → 메트릭 업데이트 → 실시간 표시
```

## 🔧 핵심 기능 구현

### 1. WebSocket 연결 관리
```typescript
useEffect(() => {
  if (evaluatorId) {
    const protocol = window.location.protocol === 'https:' ? 'wss:' : 'ws:';
    const wsUrl = `${protocol}//${window.location.host}/ws/evaluation/${evaluatorId}/`;
    
    const ws = new WebSocket(wsUrl);
    
    ws.onopen = () => {
      console.log('Progress tracking WebSocket connected');
      setWebsocket(ws);
    };
    
    ws.onmessage = (event) => {
      try {
        const message: WebSocketProgressMessage = JSON.parse(event.data);
        if (message.type === 'PROGRESS_UPDATE') {
          handleProgressUpdate(message.data);
        }
      } catch (error) {
        console.error('WebSocket 메시지 파싱 오류:', error);
      }
    };
    
    return () => ws.close();
  }
}, [evaluatorId]);
```

### 2. 진행률 메트릭 계산
```typescript
const calculateMetrics = () => {
  const totalSteps = steps.length;
  const completedSteps = steps.filter(step => step.completed).length;
  const percentageComplete = totalSteps > 0 ? (completedSteps / totalSteps) * 100 : 0;
  
  // 일관성 평균 계산
  const completedStepsWithCR = steps.filter(step => 
    step.completed && step.consistencyRatio !== undefined
  );
  const averageConsistency = completedStepsWithCR.length > 0 
    ? completedStepsWithCR.reduce((sum, step) => sum + (step.consistencyRatio || 0), 0) / completedStepsWithCR.length
    : 0;
  
  // 예상 소요 시간 계산
  const avgTimePerStep = Array.from(stepTimings.values())
    .reduce((sum, time) => sum + time, 0) / Math.max(stepTimings.size, 1);
  const remainingSteps = totalSteps - completedSteps;
  const estimatedTimeRemaining = Math.round(avgTimePerStep * remainingSteps / 60);
  
  setMetrics({
    totalSteps,
    completedSteps,
    percentageComplete,
    averageConsistency,
    estimatedTimeRemaining,
    currentPhase: getCurrentPhase()
  });
};
```

### 3. 현재 단계 구분
```typescript
const getCurrentPhase = (): string => {
  const currentStep = steps[currentStepIndex];
  if (!currentStep) return '';
  
  if (currentStep.level === 1) {
    return '주기준 비교';
  } else if (currentStep.level > 1 && currentStep.level < 99) {
    return '하위기준 비교';
  } else {
    return '대안 평가';
  }
};
```

### 4. 일관성 상태 표시
```typescript
const getConsistencyBadge = (cr?: number) => {
  if (cr === undefined) return null;
  
  let colorClass = '';
  let label = '';
  
  if (cr <= 0.05) {
    colorClass = 'bg-green-100 text-green-800';
    label = '우수';
  } else if (cr <= 0.08) {
    colorClass = 'bg-blue-100 text-blue-800';
    label = '양호';
  } else if (cr <= 0.10) {
    colorClass = 'bg-yellow-100 text-yellow-800';
    label = '허용';
  } else {
    colorClass = 'bg-red-100 text-red-800';
    label = '불량';
  }
  
  return (
    <span className={`inline-flex items-center px-2 py-1 rounded-full text-xs font-medium ${colorClass}`}>
      CR: {(cr * 100).toFixed(1)}% ({label})
    </span>
  );
};
```

## 📈 성능 최적화

### 1. 메모리 효율성
```typescript
// Map을 사용한 효율적인 타이밍 저장
const [stepTimings, setStepTimings] = useState<Map<string, number>>(new Map());

const handleProgressUpdate = (data: any) => {
  if (data.timeSpent) {
    setStepTimings(prev => new Map(prev.set(data.stepId, data.timeSpent)));
  }
};
```
