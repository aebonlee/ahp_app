#### 1. ✅ 계층 구조 문제 완전 해결
##### 문제 상황
- 상위기준과 하위기준으로 입력 후 저장하면 모두 상위기준으로 변경됨
- 백엔드에서 모든 parent_id가 null로 반환

##### 근본 원인
1. **Django Serializer 문제**:
   - `CriteriaSerializer`에서 parent 필드가 `SerializerMethodField`로 정의
   - 읽기 전용 필드라 데이터 저장 시 parent 정보가 저장되지 않음

2. **ViewSet 데이터 처리 문제**:
   - parent ID를 정수로 변환하려 했으나 실제로는 처리 불필요
   - Django ORM이 자동으로 관계 처리

##### 해결 방법
**Django 백엔드 수정 (ahp-django-service-repo)**:

1. `apps/projects/serializers.py`:
```python
class CriteriaSerializer(serializers.ModelSerializer):
    # parent 필드를 PrimaryKeyRelatedField로 변경 (쓰기 가능)
    parent = serializers.PrimaryKeyRelatedField(
        queryset=Criteria.objects.all(),
        allow_null=True,
        required=False
    )
    parent_id = serializers.SerializerMethodField(read_only=True)
    
    def to_representation(self, instance):
        """Custom representation to return parent as string ID"""
        data = super().to_representation(instance)
        if instance.parent:
            data['parent'] = str(instance.parent.id)
        else:
            data['parent'] = None
        return data
```

2. `apps/projects/views.py`:
```python
# 불필요한 parent ID 변환 로직 제거
if data.get('parent_id') and not data.get('parent'):
    data['parent'] = data['parent_id']
    data.pop('parent_id', None)
    print(f"✅ Using parent_id as parent: {data['parent']}")
```

**프론트엔드 수정 (ahp_app)**:
- `src/components/admin/CriteriaManagement.tsx`:
  - ID와 parent_id를 문자열로 통일하여 타입 불일치 방지
  - flattenCriteria 함수에서 parent_id 올바르게 설정
