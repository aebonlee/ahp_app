## #42. 프론트엔드 전용 인증 시스템 전환

**날짜**: 2026-03-07
**작업자**: Claude Opus 4.6
**유형**: refactor (아키텍처 변경)

### 배경
- Render.com의 Django REST 백엔드 및 PostgreSQL DB가 비활성/삭제됨
- JWT 기반 인증이 동작 불가 (API 서버 없음)
- 로그인 시 "Failed to fetch" 에러 발생
- 프론트엔드 단독 운영 필요

### 변경 파일 (6개, -396줄)

| 파일 | 변경 | 상세 |
|------|------|------|
| `src/services/authService.ts` | **완전 재작성** | JWT 백엔드 인증(424줄) → 프론트엔드 모의 인증(186줄). 어떤 입력이든 super_admin으로 로그인 |
| `src/hooks/useAuth.ts` | **수정** | sessionService 의존성 제거, SUPER_ADMIN_EMAIL 검증 제거, 모든 로그인을 super_admin으로 처리 |
| `src/hooks/useBackendStatus.ts` | **완전 재작성** | 백엔드 상태 체크 API 호출 제거(153줄 → 39줄). 즉시 `unavailable` + `isNavigationReady: true` 반환 |
| `src/components/AppContent.tsx` | **수정** | auto-login 백엔드 API 호출 → localStorage에서 사용자 정보 복원 |
| `src/components/auth/UnifiedAuthPage.tsx` | **수정** | 이메일 형식 검증 완화 (strict regex → 빈 값만 체크), 비밀번호 길이 검증 제거 |
| `src/services/sessionService.ts` | **수정** | 서버 로그아웃 API 호출 제거, 미사용 import(API_BASE_URL, logger) 정리 |

### 제거된 기능
- JWT 토큰 발급/갱신/만료 관리 (authService)
- 백엔드 `/api/` 헬스체크 (useBackendStatus)
- 서버 세션 검증 `validateSession()` (useBackendStatus)
- 서버 로그아웃 API 호출 (sessionService)
- 소셜 로그인 리다이렉트 (Google/Kakao/Naver — 메서드 존재하나 no-op)

### 동작 방식
1. 이메일(또는 아무 텍스트) + 비밀번호 입력
2. 입력값이 있으면 즉시 `super_admin` 역할로 로그인 성공
3. `localStorage`에 사용자 정보 저장 → 새로고침 시 세션 유지
4. 관리자 대시보드(`personal-service`)로 자동 리디렉트
